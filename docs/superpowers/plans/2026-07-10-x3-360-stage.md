# X3 360 Degree Stage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the home-page image animation with an offline, touch-controllable, continuously rotating 3D ROSMASTER X3.

**Architecture:** Assemble the public ROS X3 STL parts using their URDF transforms and export one mobile GLB. Render that GLB with a local Three.js scene hosted by a focused Android WebView wrapper; keep rotation behavior constants in pure Kotlin and expose browser diagnostics for rendering tests.

**Tech Stack:** Android Views, Kotlin, WebView, Three.js r146, GLB, Python trimesh, JUnit 4, Playwright.
