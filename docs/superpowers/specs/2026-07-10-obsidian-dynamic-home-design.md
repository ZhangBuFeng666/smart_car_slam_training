# 黑金动态首页设计说明

> 本文中的 2.5D 图片方案已由 `2026-07-10-x3-360-stage-design.md` 的真 3D 方案取代。

## 目标

将当前首页从浅色功能面板升级为黑金“数字车钥匙”主舞台，并用手册中的真实 iCar X3 产品图实现流畅的 2.5D 动态效果。现有 HTTP 控制、驾驶、任务、视觉和导航接口保持不变。

## 视觉结构

首页隐藏浅色顶部标题栏，主舞台使用炭黑背景、香槟金主操作和低亮度灰色辅助信息。真实小车位于视觉中心，背后只有雷达扫描环和细线坐标，不叠加说明卡片。主状态显示“巡逻待命”，下方保留电量、异常数和预计巡逻时间三个简洁指标。

底部导航仍提供巡逻、驾驶、任务、视觉和地图入口。首页主按钮进入导航/巡逻准备流程，不在路线和后端服务尚未配置时伪造“已经开始巡逻”的状态。急停继续使用现有控制入口和红色危险语义。

## 动态小车

小车使用《智能小车使用手册》第 5 页的 X3 产品图，经原 PDF 的透明蒙版提取为项目内 PNG。待命时小车进行幅度不超过 4dp 的缓慢上下位移和不超过 1.2 度的轻微摆动；雷达环以约 4.2 秒为一个周期旋转。用户在小车区域水平拖动时，视图产生最大 7 度的 Y 轴透视，松手后以阻尼动画回到中位。

动画在视图离开窗口时停止，回到首页时恢复，避免后台持续刷新。连接成功后状态灯与雷达线使用青绿色，未连接时使用低亮度金灰色。后续获得 GLB 模型时，可以替换该视图的绘制层，不改变首页布局和状态接口。

## 边界

本阶段不引入 Three.js、Filament、Lottie 或其他大型渲染依赖，不制作虚假的 360 度模型，不改变 Jetson 服务端协议，不实现停车场路线编排。动态效果只服务首页产品质感和状态反馈。

## 验证

运动范围、拖动限幅、雷达周期和回弹参数由纯 Kotlin 单元测试覆盖。项目必须通过 `testDebugUnitTest` 与 `assembleDebug`。若本机可找到 ADB，再安装到已连接手机检查首页无裁切、拖动与滚动手势不冲突、切换页面后动画停止。
