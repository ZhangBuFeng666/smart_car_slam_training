# X3 360 度三维展示设计

## 目标

将首页现有的单张图片透视动画替换为真实三维 ROSMASTER X3。小车围绕自身竖轴持续完成 360 度旋转，用户能够看到前、侧、后各个方向；车身位置保持稳定，不出现上下漂浮或左右摆动。

## 模型与渲染

模型采用 `rosmaster_description` 中与 ROSMASTER X3 对应的 STL 网格，包括底盘、四个麦克纳姆轮、相机和雷达。各部件按原 URDF 坐标装配后转换为单个本地 GLB，降低移动端加载和请求开销。模型资源与 Three.js 均随 APK 打包，展示不依赖外网。

Three.js 使用透明 WebGL 画布嵌入现有黑金首页主舞台。场景包含低亮度环境光、金色轮廓光、稳定的地面阴影和雷达圆环。模型中心和相机位置在加载后根据包围盒计算，保证不同屏幕比例下完整显示且不被标题、指标遮挡。

## 动画与交互

- 自动旋转只改变模型组的 Y 轴角度，默认 8 秒完成一圈，速度恒定。
- 手指横向拖动时立即暂停自动旋转，并按拖动距离修改观察角度。
- 松手后保留当前角度，短暂停顿后从该角度继续自动旋转，不回弹到正面。
- 纵向滑动交给首页滚动容器，避免三维舞台阻断页面滚动。
- 首页不可见、应用进入后台或视图被移除时暂停渲染；再次可见时继续。
- 系统关闭动画时保留手动拖动，但不自动旋转。

## Android 边界

`Vehicle3DStageView` 封装 WebView、安全的本地资源映射、生命周期和连接状态。`MainActivity` 仅创建视图并调用 `setConnected`、`onHostResume`、`onHostPause`。原有 HTTP 控制、巡逻任务、驾驶页面和底部导航不改变。

## 失败处理

模型加载前显示低亮度雷达地台；加载成功后淡入模型。WebGL 或资源加载失败时展示静态 X3 图片作为降级内容，并停止重复加载，不影响首页其他功能。

## 验收标准

1. 模型可连续展示完整前、侧、后视图，8 秒左右回到原角度。
2. 车体中心在自动旋转期间不发生可见的上下位移。
3. 横向拖动立即接管角度，松手后从当前位置继续旋转。
4. 首页滚动、底部导航和所有小车控制功能保持可用。
5. 本地浏览器检查画布非空，Android 单元测试、Lint 和 Debug 构建通过。
6. APK 覆盖安装到已连接手机，由用户确认最终视觉效果后再提交 Git。
