# iCar Web 桌面端设计

## 目标

在现有仓库中新增独立的 `web/` 项目，将 Android APP 的功能和视觉语言迁移到电脑浏览器。第一阶段面向与小车处于同一局域网的笔记本，第二阶段再增加公网访问能力。

Web 端不是新的管理后台，也不替代 Android APP。它与 APP 共用 Jetson 控制服务、摄像头视频流和 Jarvis 服务，并针对电脑横屏、鼠标和键盘重新组织布局。

## 范围

第一阶段严格对应当前 APP 的五个导航页面：

1. 巡逻
2. 驾驶
3. AI
4. 视觉
5. 导航

不新增独立的“车辆首页”或额外的“停车场巡逻”页面。Android 代码中的 `home` 在界面上对应“巡逻”。

第一阶段包含：

- 浅色与黑金双主题；
- 小车地址配置和连接状态；
- 鼠标及键盘驾驶；
- MJPEG 实时视频；
- 当前 APP 已有的任务控制；
- Jarvis 会话、任务准备和执行状态；
- 视觉与导航现有功能入口；
- 桌面横屏响应式布局。

第一阶段不包含：

- 公网账号系统；
- 云端设备网关；
- 多租户或多车辆权限管理；
- 尚未完成的真实视觉模型推理；
- 浏览器直接调用 ROS 2；
- 由 LLM 直接控制连续轮速。

## 项目结构

Web 项目位于仓库根目录：

```text
smart_car_slam_training_repo/
├── app/             Android APP
├── jetson_server/   Jetson 控制服务
├── jarvis_agent/    Jarvis AI 服务
├── web/             React Web 桌面端
└── docs/
```

`web/` 使用 React、TypeScript 和 Vite。Android APP 保持独立，两个客户端通过现有 HTTP 契约共享后端能力。

## 系统架构

Web 客户端使用统一 API 层连接两项服务：

- Jetson 控制服务 `:8000`：健康状态、底盘、移动、急停、任务、语音和摄像头；
- Jarvis 服务 `:8100`：会话、任务规划、准备、确认、执行、停止和状态轮询。

驾驶页直接消费 MJPEG 视频流。Web 客户端不直接连接串口或 ROS 2，所有硬件控制继续由 Jetson 服务负责。

第一阶段由浏览器在局域网直连小车。第二阶段增加云端网关、HTTPS、账号和设备绑定时，页面层保持不变，只替换 API 传输与认证实现。

## 页面设计

### 全局框架

桌面端采用左侧固定导航、中央主任务区和按需出现的右侧状态区。顶部统一显示产品名称、小车连接状态、当前地址和主题切换。主题切换只有一个全局入口。

布局支持 1366x768 和 1920x1080。页面主体不能依赖横向滚动，关键操作和状态不得相互遮挡。

### 巡逻

对应 APP 的 `home` 页面。保留 B2 停车场场景、巡逻状态、检查点和驾驶、AI、视觉、导航入口。停车场地图或巡逻场景作为主视觉，状态和异常信息位于辅助区域，避免卡片堆叠。

### 驾驶

摄像头是主画面，方向控制、速度预设、底盘和急停在同一屏幕内。支持鼠标按住控制以及 `W/A/S/D` 键盘控制，空格键用于急停。

### AI

复用 Jarvis 会话与任务模型。桌面端使用固定会话侧栏、对话时间线和任务卡区域，展示准备、就绪、运行、完成、失败和停止状态。危险操作仍需用户确认。

### 视觉

主区域显示摄像头及检测框，辅助区域显示类别、置信度、统计和异常事件。真实模型未接入时明确显示“训练中”或“未启动”，不生成虚假实时结果。

### 导航

地图作为主工作区，提供建图流程、初始位姿、目标点、路径规划、DWA/TEB 导航和停止操作。导航急停始终可见。

## 客户端状态

全局状态包括：

- 当前页面；
- 浅色或黑金主题；
- Jetson 地址和 Jarvis 地址；
- 小车连接状态；
- 当前驾驶状态；
- 视频流状态；
- 当前 Jarvis 会话。

主题和服务地址保存在浏览器本地存储。车辆运行状态以服务端返回结果为准，不把本地缓存当作真实硬件状态。

## 驾驶安全

- 鼠标或键盘按住时移动，松开时立即发送停止；
- 页面失焦、标签页隐藏、网络断开和驾驶组件卸载时发送停止；
- 输入框获得焦点时禁用驾驶快捷键；
- 空格急停优先于其他快捷键；
- 移动请求使用现有低延迟接口，并限制并发和重复请求；
- 小车端继续保留命令超时自动停车，前端安全逻辑不替代服务端保护；
- LLM 不能调用连续移动或任意轮速接口。

## 错误处理

- 健康检查失败时显示离线状态，并允许用户重试或修改地址；
- 视频失败时只替换视频区域为重连状态，不阻塞急停和基础控制；
- 控制请求超时时区分“结果未知”和“明确失败”，避免错误地重复移动命令；
- Jarvis 轮询失败时保留最后已知状态，并明确显示状态可能过期；
- 浏览器不支持或拦截局域网请求时显示可执行的诊断信息；
- 所有失败信息使用面向用户的描述，详细错误记录在开发日志中。

## 测试与验收

自动测试覆盖：

- API URL 和请求参数生成；
- 驾驶按下、松开、失焦和急停状态机；
- 键盘焦点保护；
- 主题与地址持久化；
- Jarvis 状态解析；
- 页面路由和关键入口；
- 视频连接状态组件。

浏览器端到端测试覆盖：

- 五页导航；
- 主题切换；
- 连接设置；
- 驾驶安全动作；
- AI 会话基本流程；
- 1366x768 与 1920x1080 的布局和无重叠检查。

真实设备验收包括：

- 成功读取小车健康状态；
- 摄像头与驾驶控制同屏；
- 鼠标和 `W/A/S/D` 均可控制；
- 松键、失焦和空格急停可靠停车；
- AI 页面读取现有会话并显示任务状态；
- 页面刷新后保留主题和小车地址。

## 第二阶段公网扩展

公网版在本地版稳定后实施。新增云端网关、HTTPS、用户登录、设备绑定、权限校验和操作审计。浏览器不直接暴露 Jetson 地址，控制命令通过受认证的网关转发。公网链路必须继续由小车端执行超时停车和安全限制。
