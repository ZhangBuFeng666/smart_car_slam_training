# Jarvis 持续控制任务状态设计

## 目标

修复自动跟随等功能启动后立即显示“已完成”、停止按钮消失的问题。持续功能必须保持运行态，直到用户通过任务卡、自然语言或全部停止明确结束。

## 任务分类

- 持续功能：`base`、`lidar`、`avoidance`、`follow`、`warning`、`camera`、`hsv`、`color_track`。
- 有界运动：按距离或时间执行的前后左右移动与转向。
- 即时操作：状态检查、停止、生成报告。

持续功能状态为 `DRAFT -> PREPARING -> READY -> STARTING -> RUNNING -> STOPPED`。持续功能启动成功后不得进入 `COMPLETED`。有界运动达到目标后继续进入 `COMPLETED`。

## 停止与同步

任务卡停止接口停止实际功能并将卡片更新为 `STOPPED`。自然语言“关闭/停止某功能”执行同一停止逻辑，并同步所有匹配的活动任务。`STOP_ALL` 同步停止所有处于 `PREPARING`、`READY`、`STARTING`、`RUNNING` 的控制任务。

重复启动同一持续功能时，返回已有活动任务，不创建重复任务卡。若停止发生在准备或启动线程执行期间，取消标志优先，后台线程不得把 `STOPPED` 覆盖回 `READY`、`RUNNING` 或 `COMPLETED`。

## APP 行为

现有 APP 对 `RUNNING` 状态持续轮询并显示停止按钮，因此无需新增状态。任务卡在停止后显示 `STOPPED`；切换会话时继续沿用现有运行任务保护。

## 验证

- 八种持续功能启动后保持 `RUNNING`。
- 任务卡停止和自然语言停止均停止进程并同步卡片。
- 重复启动返回同一任务 ID。
- 全部停止同步所有活动任务。
- 启动/停止竞争最终保持 `STOPPED`。
- 有界运动仍正常进入 `COMPLETED`。
