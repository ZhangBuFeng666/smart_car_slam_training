# 贾维斯语音回复设计

## 目标

让贾维斯在 APP 显示完整回答的同时，通过小车 USB 扬声器播报适合口语表达的简短摘要。文字回复不能等待语音播放，语音失败也不能影响聊天、控制任务或安全停止操作。

## 用户体验

- APP 继续显示完整的贾维斯回答。
- 小车自动播报同一回答的 1 至 2 句口语摘要，默认不超过 60 个汉字。
- AI 页面提供一个统一的“语音播报”开关，默认开启。
- 每条助手回复提供扬声器图标，可重新播报该条摘要。
- 多条语音按顺序播放，不允许重叠。
- 控制任务的准备、启动、完成、停止和失败使用预设短句，不额外调用模型。

## 不在本阶段实现

- 紧急播报抢占普通播报。
- 用户自定义音色、语速和音调界面。
- 流式语音合成。
- 在手机扬声器上播放。
- 语音识别或语音唤醒。

## 总体架构

一次 DeepSeek 请求同时产生两个字段：

- `reply`：APP 展示的完整回答。
- `spoken_reply`：小车播报的口语摘要。

APP 在聊天请求中携带当前语音开关状态。贾维斯生成回复后，将 `spoken_reply` 交给 Jetson 控制服务的语音队列；排队成功后立即返回聊天响应，不等待 Edge TTS 生成或音频播放结束。

```text
APP -> Jarvis /api/v1/chat
    -> DeepSeek 单次生成 reply + spoken_reply
    -> Jetson /speech/enqueue
    -> 单工作线程语音队列
    -> Edge TTS
    -> USB 扬声器
```

## API 设计

### 贾维斯聊天请求

在现有请求中增加可选字段，保持旧客户端兼容：

```json
{
  "message": "现在看到了什么？",
  "context": {},
  "speech_enabled": true
}
```

`speech_enabled` 缺省为 `false`，防止旧客户端升级服务后突然自动播报。新版 APP 明确传递本地开关状态。

### 贾维斯聊天响应

```json
{
  "reply": "完整的文字回答",
  "spoken_reply": "适合播报的简短回答",
  "speech": {
    "state": "queued",
    "request_id": "唯一标识"
  }
}
```

`speech.state` 允许值为：

- `queued`：已进入小车语音队列。
- `disabled`：用户关闭了自动播报。
- `unavailable`：控制服务或语音队列不可用。
- `skipped`：没有可播报内容。

语音不可用时，聊天接口仍返回成功，APP照常显示 `reply`。

### Jetson 语音入队接口

新增 `POST /speech/enqueue`：

```json
{
  "text": "我看到前方通道暂时畅通。",
  "source": "jarvis",
  "request_id": "唯一标识"
}
```

接口仅校验并入队，不等待播放完成。现有 `POST /speak` 保留，兼容当前通知和手动测试。

## 模型输出

DeepSeek 普通问答改为 JSON 输出：

```json
{
  "reply": "完整回答",
  "spoken_reply": "口语摘要"
}
```

提示词要求：

- `spoken_reply` 使用自然中文口语。
- 控制在 1 至 2 句、60 个汉字以内。
- 不朗读 Markdown 标记、URL、代码、密钥、长坐标或长数字序列。
- 涉及危险或失败时必须保留关键警告。
- 不虚构视觉、传感器或任务状态。

解析失败时不进行第二次模型请求。服务保留完整文字，并按中文句号、问号或感叹号提取第一句作为播报降级；降级文本仍执行长度和敏感内容过滤。

## 固定控制回复

自动跟随、避障、警卫、目标追踪和底盘移动等控制命令可能不经过 DeepSeek。此类响应由贾维斯提供确定性播报文本，例如：

- `正在完成启动前准备。`
- `自动跟随已准备好，等待启动。`
- `自动跟随已启动。`
- `自动跟随已停止。`
- `任务启动失败，请查看详细信息。`

任务进度轮询不得重复播报同一状态。仅状态发生关键变化时播报一次。

## 语音队列

Jetson 控制服务维护一个进程内、单消费者 FIFO 队列：

- 同一时刻只生成和播放一条语音。
- `request_id` 在短时间窗口内去重，避免 APP 重试造成重复播报。
- 每条文本在入队前去除空白并限制长度。
- 服务停止时终止工作线程并清理临时音频文件。
- Edge TTS 失败后沿用现有 `espeak-ng` 降级，但失败不会让队列线程退出。

本阶段不实现优先级和抢占。紧急通知仍可调用现有同步接口；后续通知分级设计会统一处理抢占策略。

## APP 交互

- AI 页面标题区域增加一个扬声器图标开关，不新增卡片。
- 开启状态表示后续回复自动播报，关闭状态只影响自动播报，不影响手动重播。
- 助手消息气泡旁增加扬声器图标；点击后将保存的 `spoken_reply` 发送到 `/speech/enqueue`。
- APP 会把 `spoken_reply` 与消息一起保存，重新打开历史会话后仍可重播。
- 入队失败只显示轻量状态提示，不插入新的错误气泡，也不重复请求 LLM。

## 错误处理

- DeepSeek 结构化输出异常：保留全文，使用本地第一句降级。
- Jetson `8000` 不可达：返回 `speech.state=unavailable`，聊天成功。
- Edge TTS 断网：尝试 `espeak-ng`，仍失败则记录日志并继续下一条。
- USB 音频设备缺失：记录明确设备错误，不切换到不确定的 HDMI 输出。
- APP 重复点击：依靠请求 ID 和本地点击节流防止连续重复播放。

## 测试与验收

### 自动化测试

- DeepSeek 双字段响应解析及格式异常降级。
- `spoken_reply` 长度、敏感内容和第一句提取。
- 聊天请求的语音开关兼容行为。
- 控制服务语音队列顺序、去重和异常后继续消费。
- 固定控制状态只播报一次。
- APP 开关持久化、请求字段、消息保存和重播请求。

### 小车实测

1. 短问答：全文显示并完整播报短回答。
2. 长问答：APP 显示全文，小车只播报 1 至 2 句摘要。
3. 连续发送两条问题：语音顺序播放且不重叠。
4. 关闭开关：文字正常显示，小车不自动播报。
5. 点击历史消息扬声器：重新播报已保存摘要。
6. 临时断开网络：文字错误行为清晰，语音线程不崩溃。

## 完成标准

- 普通回复和控制任务回复都能产生可播报短句。
- APP 文字响应不等待音频播放结束。
- 连续语音没有重叠或明显重复。
- 关闭自动播报后不产生后台语音。
- TTS 或扬声器故障不影响贾维斯聊天和小车控制。
